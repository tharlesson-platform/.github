# Security Policy

This organization uses a shared security policy for all actively maintained repositories.

## Reporting a Vulnerability

Please do not open public issues for suspected vulnerabilities.

- Email: tharlesson@msn.com
- Prefer a GitHub security advisory in the affected repository when that option is available.

Include the affected repository, impact, reproduction steps, and any logs or screenshots that are safe to share.

## Response Expectations

- Initial acknowledgement target: 3 business days.
- Triage and remediation planning happen after the report is validated.

## Disclosure

Please give reasonable time for investigation and remediation before public disclosure.
